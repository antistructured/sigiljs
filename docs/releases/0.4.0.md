# 0.4.0 Release Notes

**Release theme:** Public API stabilization  
**Package:** `@weipertda/sigiljs`  
**Date:** 2026-06-21

## Highlights

This release stabilizes the public API surface of SigilJS before any future package extraction. The package remains a single distributable: `@weipertda/sigiljs`.

## Added

- Stabilized public API documentation in `docs/api.md`
- Public export audit: `docs/internal/public-export-audit.md`
- Experimental API index: `docs/experimental.md`
- 0.4.0 readiness report: `docs/internal/0.4.0-readiness.md`
- Public API tests: `tests/public-api.test.js` and `tests/public-api-surface.test.js`
- Projection API freeze documentation: `docs/projections.md` and `docs/internal/projection-api-freeze.md`
- Alias policy documentation: `docs/sigils.md`

## Changed

- Clarified `sigil()` as the preferred public API, with `Sigil`\`template\` as the template alternative
- Documented `S` and `T` as legacy aliases, not recommended for new code
- Classified `httpContract()` as experimental
- Classified `toFormConstraints()` as experimental
- Confirmed `mock()` and `cases()` as stable testing helpers
- Hardened projection API behavior with explicit input/output contracts

## Fixed

- Tests updated to import from the public entry point only
- Removed accidental internal-dependency imports from public API tests
- Preserved backward compatibility for legacy aliases in existing tests

## Notes

- No packages were split
- No core systems were rewritten
- `httpContract()` and forms projection remain exported but are not stable boundaries
- Future `@sigil/http` extraction is not implied in this release
